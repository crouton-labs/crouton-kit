# Grove Headless

**Arguments:** $ARGUMENTS

The script manages dev servers as supervised background processes with grove-instance-safe log isolation (`/tmp/grove/<path-slug>/`). Because the files live under `.claude/`, every instance planted from this source automatically inherits them.

## Step 3: Discovery

Launch **three parallel Explore agents** to build a complete picture. Each agent returns a concise structured report.

### Agent 1: Service inventory

Find every runnable service and how it starts.

- Read `.claude/grove/config.json` if present — its `ports` block enumerates services and the `repos` / `copyFromSource` hints which subdirectories host them.
- Walk every `package.json` in the repo (including subdirectories that look like apps/services). For each, extract: the `dev`, `start:dev`, `start`, `app:start*` scripts; the package manager (presence of `pnpm-lock.yaml`, `yarn.lock`, `bun.lockb`, fallback npm); any `PORT` references in those scripts.
- Check `docker-compose*.yml` for services that should be started via docker rather than a long-running node process.
- Note any Electron/Tauri apps (`electron`, `electron-builder`, `CDP_PORT` in `.env`) — these need a rebuild-then-launch pattern rather than plain dev.
- Note any service with a multi-stage bootstrap (libs that must be built before start, codegen, DB migration) — these warrant custom pre-start logic.

**Report per service:** name (short alias like `core`, `gateway`, `agent`), working directory (relative to repo root), start command, port env var + default port (read from `<dir>/.env` with `PORT=` or similar), package manager, and any bootstrap steps.

### Agent 2: Port env convention

Ports must be read from `.env` at runtime so the same script works in every grove instance (each instance has its own patched `.env`). For each service identified by Agent 1:

- What env var name holds its port? (`PORT`, `CORE_PORT`, `CDP_PORT`, etc.)
- What `.env` file is it in? (`<dir>/.env`, root `.env`, etc.)
- What's the default if the env var is missing?

Flag any service whose port is hardcoded in source rather than read from `.env` — those should be fixed before headless.sh can correctly isolate instances.

### Agent 3: Bootstrap & dependency ordering

Some services can't be started cold. Report any:

- Codegen steps that must run before start (`prisma generate`, GraphQL codegen, TypeScript compilation of shared libs).
- Built artifacts that grow stale and cause silent failures (`dist/` older than source → restart needed).
- Electron-style rebuild loops (source changes require `npm run build:*` before `app:start`).
- Services whose build output is consumed by another service (ordering constraints).
- Database prerequisites (Postgres container must be up, migrations run).

**Report:** which services have bootstrap concerns, what the steps are, and whether they should run every launch or only once.

### Supervisor core (paste verbatim)

```bash
#!/usr/bin/env bash
# Headless dev server manager — generated by /grove:headless
# Edit the "Project-specific dispatch" section below; the utilities above are reusable.

set -uo pipefail

# ─── Log directory (grove-instance-safe) ─────────────────────────────────────
# Path slug ensures two instances on disk get separate log dirs.

REPO_ROOT="$(cd "$(dirname "$0")/../.." && pwd)"
SLUG="$(echo "$REPO_ROOT" | sed 's|^/||; s|/|-|g')"
LOG_DIR="/tmp/grove/${SLUG}"
mkdir -p "$LOG_DIR"

log_path() { echo "${LOG_DIR}/$1-dev.log"; }

# ─── Process & port utilities ────────────────────────────────────────────────

wait_for_port_free() {
    local port=$1 i=0
    while lsof -i :$port -sTCP:LISTEN >/dev/null 2>&1 && [ $i -lt 20 ]; do
        sleep 0.5; i=$((i + 1))
    done
    [ $i -lt 20 ]
}

kill_on_port() {
    local pid=$(lsof -i :$1 -sTCP:LISTEN -t 2>/dev/null)
    if [ -n "$pid" ]; then
        kill $pid 2>/dev/null
        wait_for_port_free $1 || kill -9 $pid 2>/dev/null
        wait_for_port_free $1
    fi
}

kill_process_tree() {
    local pid=$1
    kill -- -$(ps -o pgid= -p $pid | tr -d ' ') 2>/dev/null || kill $pid 2>/dev/null
    pkill -P $pid 2>/dev/null
}

# Returns "free", "ours:<pid>", or "other".
# Checks the pid AND its parent — some dev servers (next-server) don't show the repo path
# in their own argv but their pnpm/npm parent does. Missing this causes false "other" verdicts
# and grove instances refusing to start services they actually own.
port_status() {
    local port=$1 dir=$2
    local pid=$(lsof -i :${port} -sTCP:LISTEN -t 2>/dev/null | head -1)
    if [ -z "$pid" ]; then echo "free"; return; fi
    local pattern="${REPO_ROOT}/${dir}"
    local ppid=$(ps -o ppid= -p "$pid" 2>/dev/null | tr -d ' ')
    if ps -o command= -p "$pid" 2>/dev/null | grep -q "$pattern" || \
       { [ -n "$ppid" ] && ps -o command= -p "$ppid" 2>/dev/null | grep -q "$pattern"; }; then
        echo "ours:$pid"
    else
        echo "other"
    fi
}

kill_service_processes() {
    local scoped="${REPO_ROOT}/$1"
    for pid in $(pgrep -f "${scoped}.*pnpm.*dev" 2>/dev/null) \
               $(pgrep -f "${scoped}.*npm.*dev" 2>/dev/null) \
               $(pgrep -f "${scoped}.*pnpm.*start:dev" 2>/dev/null) \
               $(pgrep -f "${scoped}/node_modules/.bin" 2>/dev/null); do
        kill_process_tree "$pid"
    done
    sleep 0.5
}

stop_service() {
    kill_service_processes "$1"
    kill_on_port "$2"
}

# Supervised restart with exponential backoff + port health watchdog.
# - Process crash: detected via wait, restarted with backoff
# - Zombie process (alive but port dead): watchdog kills after 30s, triggers restart
# - TERM=dumb prevents tsc --watch from emitting clear-screen sequences / null bytes
start_service() {
    local dir=$1 cmd=$2 log=$3 port=${4:-} max_retries=${5:-5}
    nohup bash -c '
        trap "cleanup" TERM INT
        dir="$1"; cmd="$2"; max_retries="$3"; port="$4"
        backoff=2; max_backoff=60; stable_threshold=30; retries=0
        child_pid=""; watchdog_pid=""

        cleanup() {
            [ -n "$watchdog_pid" ] && kill "$watchdog_pid" 2>/dev/null
            [ -n "$child_pid" ] && kill "$child_pid" 2>/dev/null
            exit 0
        }

        while true; do
            start_time=$(date +%s)
            echo "[$(date "+%Y-%m-%d %H:%M:%S")] START: $cmd (attempt $((retries + 1)))"
            (cd "$dir" && TERM=dumb eval "$cmd") &
            child_pid=$!

            if [ -n "$port" ]; then
                _wd_child=$child_pid; _wd_port=$port
                (
                    sleep 60
                    fail_count=0
                    while kill -0 "$_wd_child" 2>/dev/null; do
                        if lsof -i :"$_wd_port" -sTCP:LISTEN >/dev/null 2>&1; then
                            fail_count=0
                        else
                            fail_count=$((fail_count + 1))
                            if [ "$fail_count" -ge 6 ]; then
                                echo "[$(date "+%Y-%m-%d %H:%M:%S")] WATCHDOG: port $_wd_port unresponsive for 30s — killing pid $_wd_child"
                                kill "$_wd_child" 2>/dev/null
                                sleep 2
                                kill -0 "$_wd_child" 2>/dev/null && kill -9 "$_wd_child" 2>/dev/null
                                break
                            fi
                        fi
                        sleep 5
                    done
                ) &
                watchdog_pid=$!
            fi

            wait "$child_pid" 2>/dev/null
            exit_code=$?
            child_pid=""
            [ -n "$watchdog_pid" ] && kill "$watchdog_pid" 2>/dev/null; watchdog_pid=""

            run_duration=$(( $(date +%s) - start_time ))
            if [ "$run_duration" -ge "$stable_threshold" ]; then
                backoff=2; retries=0
            else
                retries=$((retries + 1))
            fi
            echo "[$(date "+%Y-%m-%d %H:%M:%S")] EXITED: code=$exit_code ran=${run_duration}s (retry $retries/$max_retries)"
            if [ "$retries" -ge "$max_retries" ]; then
                echo "[$(date "+%Y-%m-%d %H:%M:%S")] GAVE UP: $max_retries consecutive fast crashes (< ${stable_threshold}s each)"
                break
            fi
            echo "[$(date "+%Y-%m-%d %H:%M:%S")] RESTART: in ${backoff}s..."
            sleep "$backoff"
            backoff=$(( backoff * 2 > max_backoff ? max_backoff : backoff * 2 ))
        done
    ' _ "$dir" "$cmd" "$max_retries" "$port" > "$log" 2>&1 &
}

# Start a service only if its port is free (or already ours).
ensure_service() {
    local name=$1 dir=$2 port=$3 cmd=$4
    local status=$(port_status "$port" "$dir")
    case "${status%%:*}" in
        free)
            kill_service_processes "$dir"
            echo "Starting $name → $(log_path "$name")"
            start_service "${REPO_ROOT}/${dir}" "$cmd" "$(log_path "$name")" "$port"
            ;;
        ours)  echo "✓ $name already on :${port}" ;;
        other) echo "⚠ Port ${port} in use by another environment — skipping $name" ;;
    esac
}

read_env_port() {
    # read_env_port <subdir> <VAR_NAME> <default>
    local port=$(grep -m1 "^$2=" "${REPO_ROOT}/$1/.env" 2>/dev/null | cut -d= -f2 | tr -d '"'"'"' ')
    echo "${port:-$3}"
}

ARGS="$*"

echo "Logs: $LOG_DIR"
```

### Project-specific dispatch (synthesize per project)

Below the core, emit:

1. **Port resolution block** — one `read_env_port` call per service, assigning to a shell variable (`CORE_PORT=$(read_env_port northlight-core PORT 3168)`).
2. **`should_run` filter** — matches user args against service aliases. Use this pattern:
   ```bash
   should_run() {
       [[ "$ARGS" =~ all ]] || ! [[ "$ARGS" =~ <alias1>|<alias2>|... ]] || [[ "$ARGS" =~ $1 ]]
   }
   ```
   The aliases in the regex are the full set; the alternation means "if user named any service, treat it as opt-in mode".
3. **Stop / restart block** — guarded by `[[ "$ARGS" =~ stop|restart ]]`. One `stop_service` call per service. Exit after if `stop`.
4. **Start block** — one `ensure_service` (or custom start with bootstrap) call per service, each guarded by `should_run <alias>`.
5. **Bootstrap-heavy services** — for anything Agent 3 flagged, write a bespoke block rather than `ensure_service` (see vault-service pattern in northlight for reference: gate behind `port_status`, run build steps, then `start_service`).

Make the script executable:

```bash
chmod +x <target-path>/.claude/scripts/headless.sh
```
